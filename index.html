Medical Chatbot Hack

It is crucial that medical staff have fast access to patient data. This Python Project makes patient data retrieval efficient, by developing a Google Assistant Chatbot to access records. The application uses Google Dialogflow to extract key information from user input. A response is then computed by querying the FHIR API. I individually completed this project in a hackathon and my chatbot was in the top 25 out of 116 submissions. It is now published officially with the Great Ormond Street Hospital.

NHS Project

The goal of this C# based project was to develop a search system for medical policies in the NHS. An interactive chatbot was designed for the user interface of the search system using the Azure Bot Framework and LUIS. Additionally to make the search system faster for managing a large database of documents, Azure Databricks was utilised. In the end we found that for 88% of sample client queries the system returned the relevant policy in the top 5 results.

Sudoku Scanner & Solver

The goal of this Python based project was to develop a system that scans a sudoku puzzle via the webcam and returns the solution to the user. The first step involves extracting the sudoku board from the captured image via Image Processing. The numbers on the sudoku board are then recognised by a Machine Learning model I trained using the MNIST dataset. The sudoku puzzle is then solved via the backtracking algorithm and the solving process is visualised using the pygame library.

Movie Recommendation System

The goal of this Python based project was to develop a system that can take a movie title and return similar movies available on TV, Amazon & Netflix. The application uses cosine similarity and the Movie Lens dataset to identify similar movies. These similar movies are then found by web scraping TV schedules via Beautiful Soup. Similar movies on Netflix & Amazon are extracted from the Kaggle datasets. The IMDB & OMDb APIs are then called to extract movie information to display to the user.

Shell Reconstruction

The aim of this Java based project was to develop a shell similar to the terminal available on computers. To parse in the inputted command the ANTLR library was utilised. We then produced a valid output for the inputted user command with the help of the Visitor, Decorator and Singleton design patterns. By the end, our shell passed 90.77% of acceptance tests (provided by professor) and supported complex commands such as the pipe operator, command substitution and commands involving regular expressions.

Equical

I am currently developing a revision management website alongside two other students using Django and Python. The user inputs topics they have studied throughout the day and the application recommends when to revisit the topic for maximum retention. These results are added automatically to the user’s Google Calendar. As Co-Tech lead of the project I had a key role in developing the sign in system. I utilised the Google Calendar API and Firebase to sign in the user via Gmail and authorise permissions to access the user’s Google Calendar.

American Express Hackathon

The goal of this Java project is to develop a mobile application that connects customers with small businesses so they can complete their list of errands for the day. To determine the best route for the user, the Google Maps API was used. Once the itinerary for the day was determined the user can explore reviews of places (developed via the Yelp API) and add their experiences of visiting the small businesses to a collection of memories. This app was developed as group project at the American Express Hackathon at UCL.
